# RAK-03: Evolution — History & Future

> **"Bahasa yang tidak berevolusi adalah bahasa yang mati. TypeScript berevolusi tiap tahun."**

Rak ketiga mendokumentasikan perjalanan `tsc` dari versi ke versi. Bukan sekadar changelog — ini adalah analisis *mengapa* sebuah fitur ditambahkan, masalah apa yang mendorongnya, dan bagaimana setiap rilis membentuk cara kita berpikir tentang sistem tipe.

## Struktur Sub-Rak

| Sub-Rak | Slug | Konten |
|:---|:---|:---|
| **SR-01** | `early-era` | TypeScript 1.x – 2.x: Kelahiran, `strictNullChecks`, `readonly`, awal *Control Flow Analysis* |
| **SR-02** | `growth-era` | TypeScript 3.x – 4.x: `unknown`, *Variadic Tuple Types*, Template Literal Types, `infer`, *Conditional Types* |
| **SR-03** | `modern-era` | TypeScript 5.x: `const` Type Parameters, *Decorator* (Stage 3), `using` keyword, *Satisfies* operator |
| **SR-04** | `roadmap` | Fitur yang sedang diusulkan di TC39 / TypeScript GitHub, dan arah masa depan type system |

## Mengapa Ini Penting?
Mengetahui riwayat fitur mencegah kamu menggunakan workaround lama yang sudah ada solusi modern-nya, dan membantu membaca *breaking changes* dengan tenang.
