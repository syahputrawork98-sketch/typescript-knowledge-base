# BK-01: Philosophy — Why Static Typing?

> *"Tipe bukan sekadar anotasi — ia adalah kontrak antara developer hari ini dan developer masa depan."*

**Unit ini tidak membutuhkan Lab Praktis/Visualisasi karena bersifat penjelasan sejarah/konsep naratif.**

## Gambaran Buku Ini
Buku pertama RAK-01 membahas pertanyaan mendasar: **mengapa kita membutuhkan static typing?** Ini bukan pertanyaan teknis — ini pertanyaan filosofis tentang bagaimana manusia membangun perangkat lunak berskala besar secara andal.
